# Client2 — HR-вакансия кассира в VK

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Короткий текст: HR хотят разместить рекламу вакансии кассира в VK. Бюджет до 50 000 ₽.

## 2. Что получилось в медиаплане

Один лист `ВК-Реклама`. Одна основная строка: объект “Вакансия”, площадка VK Реклама, расположение LeadAds, таргетинг Москва/МО 18+, формат CPC. Есть итоговые строки: бюджет без НДС, с НДС, АК, CTR, клики, прогноз лидов.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| Вакансия/HR | класс кампании HR / Lead generation |
| VK из брифа | единственная площадка, без альтернативного сплита |
| Бюджет 50 000 | поставлен как стоимость с НДС |
| Формат не указан | выбран LeadAds как наиболее прямой формат под отклик на вакансию |

## 4. Что медиапланер добавил сам

- Гео Москва/МО добавлено экспертно, в брифе его нет.
- Период “Октябрь” добавлен экспертно.
- CTR/CPC/лиды взяты из бенчей/справочника, не из брифа.

## 5. Правила, которые надо вынести в MVP

- Если HR + VK + бюджет до 100k — делать минимальный план из одной строки VK LeadAds.
- Малый бюджет не дробить на несколько площадок.
- Если нет гео/периода — не молча подставлять, а показывать warning и разрешать быстрый default.
- Для HR считать CPC и прогноз лидов, а не охват как главный KPI.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
