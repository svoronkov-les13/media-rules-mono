# Client7 — импортные товары в доставке, 360 + digital + блогеры

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Кампания по продвижению зарубежных товаров в доставке. Нужно рассказать о товарах высокого ценового сегмента/скидках, мотивировать попробовать. Каналы 360: ТВ, OOH, радио, digital, SMM, блогеры. Период примерно 19.08–06.10. Фокус гео: Москва/СПб, усиление Самара/НН/Екатеринбург/Тюмень. Есть описание ЦА и конкурентного контекста.

## 2. Что получилось в медиаплане

Листы: `МП Digital`, `гео`, `cover`, Ozon, Avito, VK Ads, Weborama, Telegram Ads, MTS, Hybrid, Bidease, Everest, интересы, подборки Telegram/VK. Есть cover с агрегированными KPI и бюджетом по городам. Основной план содержит retail media, programmatic, social, influencer-блоки.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| 360-кампания | digital-часть превращена в широкий медийный микс |
| Фокус/усиление по городам | становится бюджетным распределением в cover |
| Импортные товары/доставка | ведёт к retail media, marketplace и поведенческим интересам |
| Блогеры/SMM | становятся отдельными подборками TG/VK |

## 4. Что медиапланер добавил сам

- Добавлены programmatic-площадки и data providers.
- Сформировано распределение бюджета по городам, которого нет как готовой таблицы в брифе.
- Подобраны интересы/таргетинги из описания аудитории.
- Появляется отдельный executive summary/cover для клиента.

## 5. Правила, которые надо вынести в MVP

- Если бриф 360, MVP должен строить только digital-медиаплан и явно отделять out-of-scope каналы.
- Для доставки/retail использовать Ozon/Avito/VK/programmatic/Telegram как кандидатов.
- Если заданы фокусные и поддерживающие города — делать бюджетную матрицу по городам.
- Influencer-подборки хранить отдельными таблицами с тематикой, форматом и ценой.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
