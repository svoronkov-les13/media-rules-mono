# Client3 — бюджеты по объектам, контекст

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Короткое письмо: клиент прислал бюджеты по объектам на сентябрь, нужно подготовить медиаплан. Есть список объектов NAME1–NAME6/NAME5 и общий бюджет 22,5 млн ₽.

## 2. Что получилось в медиаплане

Два листа: `Контекст` и `Итого`. На каждый объект заведена строка Яндекс.Директ: поиск + РСЯ, ключевые слова/интересы, текст/ТГБ/баннеры/видео, формат CPC. Лист `Итого` подтягивает суммы с НДС и с НДС+АК по объектам.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| Список объектов | каждый объект становится отдельной строкой медиаплана |
| Бюджет объекта | становится бюджетом строки |
| Сентябрь | ставится как срок оказания услуг |
| Тип площадки не указан | выбран стандартный контекстный сценарий Яндекс.Директ |

## 4. Что медиапланер добавил сам

- В брифе есть неоднозначность: повторяется NAME5 и сумма требует проверки.
- CPC по объектам различается — значения взяты из бенчей/истории.
- Семантика не приложена отдельными вкладками, используется типовая формулировка “ключевые слова, интересы”.

## 5. Правила, которые надо вынести в MVP

- Если в брифе перечислены объекты с бюджетами — сохранять объектную структуру, не оптимизировать бюджет без разрешения.
- Автоматически проверять дубли объектов и совпадение суммы с итогом.
- Для “бюджеты по объектам” выбирать шаблон `Контекст + Итого`.
- Если канал не указан, но задача типовая performance/контекстная — предлагать Яндекс.Директ как default с пометкой “требует подтверждения”.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
