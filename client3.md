# Client3 — бюджеты по объектам

<div class="page-meta">Pair: <code>Бриф_client3</code> → <code>МП_client3</code></div>

## Summary

| Параметр | Значение |
|---|---|
| Тип задачи | Multi-object context |
| Вход | TXT-бриф: список объектов с бюджетами на сентябрь, общий бюджет 22,5 млн ₽. |
| Выход | Excel: Контекст + Итого. Каждый объект — строка Яндекс.Директ, поиск/РСЯ, CPC, итоговые суммы. |
| Ключевое правило | Если клиент дал бюджеты по объектам, структура объектов сохраняется без автоматической оптимизации. |
| Главный риск | Повтор NAME5 и несоответствия суммы должны ловиться валидатором. |

## Brief fields

| Поле | Значение | Источник | Действие в системе |
|---|---|---|---|
| Объекты | NAME1…NAME6/NAME5 | Brief | Строки медиаплана |
| Бюджеты | по каждому объекту | Brief | Бюджет строки |
| Период | Сентябрь | Brief | Срок оказания услуг |
| Канал | не указан | Planner default | Яндекс.Директ |
| KPI | не указан | Missing | Warning |

## Media plan structure

| Лист / блок | Назначение |
|---|---|
| Контекст | Строки по объектам |
| Итого | Ссылки на суммы с НДС и с НДС+АК |

## Mapping rules

| Из брифа | В медиаплан |
|---|---|
| Объект | отдельная строка |
| Бюджет объекта | бюджет строки |
| Сентябрь | период строки |
| Канал не указан | типовой Яндекс.Директ |

## Planner assumptions

- CPC по объектам.
- Стандартные форматы поиск/РСЯ.
- Лист итогов.

## MVP rules

- Сохранять объектную структуру.
- Проверять дубли и сумму.
- Шаблон: Контекст + Итого.
- Default-канал помечать как допущение.
