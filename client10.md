# Client10 — короткий бриф “по бенчам и прошлым РК”

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Короткое сообщение: нужен план на август для Client10. Бюджет 22 289 884 ₽ без НДС. Площадки подобрать согласно бенчам и рекомендациям по результатам прошедших РК.

## 2. Что получилось в медиаплане

Листы `МП` и `ГЕО`. Шапка: Client10, период 01.08–31.08, объём ЦА 62 млн, охват ЦА формулой. В плане есть Video и Display, первая строка MTS Multi-roll/in-stream, CPM, desktop+mobile, все регионы присутствия. Вкладка `ГЕО` содержит регионы/области.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| Бюджет без НДС | используется как базовое ограничение без деления на 1.2 |
| Август | становится периодом 01.08–31.08 |
| “по бенчам и прошлым РК” | означает выбор площадок из исторических рекомендаций |
| Гео не в брифе | взято из справочника регионов присутствия |

## 4. Что медиапланер добавил сам

- Почти вся логика выбора площадок не содержится в брифе.
- Нужны исторические кампании Client10: факты, рекомендации, бенчи.
- Объём ЦА и гео подтянуты из внешнего справочника.
- Система без базы прошлого опыта сможет сделать только шаблонный, а не экспертный план.

## 5. Правила, которые надо вынести в MVP

- Если в брифе есть “по бенчам/прошлым РК” — искать ближайшую историческую кампанию клиента и использовать её как референс.
- Если исторических данных нет — ставить blocker/warning и просить загрузить бенчи.
- Для коротких брифов явно показывать список допущений.
- Бюджет “без НДС” не преобразовывать как “с НДС”; режим бюджета должен храниться явно.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
