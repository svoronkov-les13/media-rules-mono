# Client4 — франшиза/кредит, performance на заявку

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Запуск кредитного продукта для франчайзи. Нужно вести на лендинг и форму заявки. Бюджет 1 млн ₽ с НДС, период октябрь, гео РФ. Обязателен расчёт CPL. Целевое действие — заявка в форме.

## 2. Что получилось в медиаплане

Листы: `МП`, две вкладки семантики Яндекс.Директ, минус-фразы, источники BigData, таргетинг TG Ads. В медиаплане: Яндекс Поиск/Мастер кампаний, РСЯ, BigData-сбор номеров, Telegram по каналам/BigData/MTS. Есть клики, показы, CTR, CR, конверсии и CPL.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| CPL/заявка | становится главным прогнозным показателем |
| Франшиза/пекарня | порождает семантические группы и минус-фразы |
| Гео РФ | переносится в настройки кампании |
| Бюджет 1 млн с НДС | распределяется между Яндекс, BigData и Telegram |

## 4. Что медиапланер добавил сам

- Добавлены дополнительные каналы Telegram и BigData, которых клиент явно не перечислял.
- Сформированы ключевые фразы по “франшиза”, “пекарня”, “кредит на франшизу”.
- CR по каналам взят из бенчей/экспертной оценки.
- Добавлен контур лидогенерации за пределами классического контекста.

## 5. Правила, которые надо вынести в MVP

- Если есть “CPL”, “заявка”, “форма”, “лендинг” — классифицировать как performance / lead generation.
- Для нишевого B2B/franchise performance создавать семантические вкладки и минус-фразы.
- В медиаплане обязательно считать CR, количество конверсий и CPL.
- Каналы расширения: РСЯ, Telegram Ads, BigData — только если бюджет позволяет и есть бенчи.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
