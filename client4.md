# Client4 — франшиза/кредит: заявки и CPL

<div class="page-meta">Pair: <code>Бриф_client4</code> → <code>МП_client4</code></div>

## Summary

| Параметр | Значение |
|---|---|
| Тип задачи | Performance / Lead generation |
| Вход | DOCX-бриф: кредитный продукт для франчайзи, лендинг, форма заявки, РФ, октябрь, 1 млн ₽ с НДС, обязателен CPL. |
| Выход | Excel: МП + семантика Яндекс + минус-фразы + BigData + TG Ads. Расчёты кликов, показов, CR, лидов, CPL. |
| Ключевое правило | CPL-запрос требует performance-шаблон с семантикой, CR и расчётом конверсий. |
| Главный риск | CR и CPL без историки являются экспертными бенчами, а не ML-прогнозом. |

|---|---|

| Лендинг/форма | performance-сценарий |

## Planner assumptions

- Telegram и BigData.
- CR по каналам.
- Семантические группы и минус-фразы.

## MVP rules

- CPL/заявка/форма → performance.
- Для нишевого B2B делать семантику и минус-фразы.
- Обязательно считать CR, лиды, CPL.
- Расширяющие каналы добавлять только с бенчами.
