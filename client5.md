# Client5 — банковская карта, Яндекс/VK/семантика

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

Нужно подготовить МП по зарплатной карте с кешбэком. Период 01.09–30.09.2025. Бюджет 500 000 ₽ с НДС и АК. Обязательные каналы: VK, Яндекс с супергео по 15 адресам, ремаркетинг, тематические/корпоративные группы VK. ЦА B2B и B2C. Цены брать из спецификации по договору.

## 2. Что получилось в медиаплане

Листы: `MediaPlan`, семантика бренд, B2B, B2C, а также РБК/Kommersant. В плане: Яндекс брендовый поиск, небрендовый поиск, РСЯ с супергео/интересами/ремаркетингом, VK пост с кнопкой, карусель и другие форматы. Расчёты ведутся с учётом НДС/АК.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| B2B/B2C ЦА | разделена на отдельные семантические вкладки |
| Яндекс + супергео | становится строками Поиск/РСЯ с геотаргетингом |
| VK варианты | превращены в несколько форматов VK |
| Цены из спецификации | должны ограничивать справочник ставок и площадок |

## 4. Что медиапланер добавил сам

- Добавлены брендовые и небрендовые группы запросов.
- Посадочная ещё не готова — это warning, но план считается.
- Добавлены медийные/PR-площадки РБК/Kommersant для проверки релевантности.
- Нужен справочник договорных спецификаций.

## 5. Правила, которые надо вынести в MVP

- Финансовый продукт с поиском — всегда разделять семантику на бренд/небренд/B2B/B2C.
- Если указано “цены из спецификации” — запрещать ставки вне договорного справочника.
- Если посадочная не готова — считать черновик, но помечать риск для CPL/конверсий.
- Для супергео хранить адреса/радиус отдельной структурой, не только текстом в строке.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
