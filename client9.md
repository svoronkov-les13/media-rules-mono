# Client9 — PPTX, OLV/banner, device cap Smart TV

[← На главную](index.html) · [Сводная матрица](matrix.html)

## 1. Что было в брифе

PPTX содержит несколько блоков. Есть имиджевая кампания: период 1 июня–6 июля, OLV 25/15 сек + баннеры, устройства Smart TV/Mobile/Desktop, Smart TV не более 15%, бюджет 11 789 444 ₽. Также есть блок “Инаут”: 19 мая–30 июня, OLV+баннеры, бюджет 28 782 579 ₽. Заданы ЦА-сегменты, бенчи CPM 170–220 и CPC 30–40, критерии оценки МП.

## 2. Что получилось в медиаплане

Листы: `Свод`, `МП Инаут`, таргетинги, площадки Segmento/Ozon/Яндекс/Bidease/Byyd и факт-лист. `МП Инаут` содержит video/display строки, CPM закупку, устройства desktop+mobile, гео, таргетинги, множители наценок/сезонности/гео/частоты/пола/возраста/формата. Есть свод по периодам/бюджетам.

## 3. Сопоставление БРИФ → МЕДИАПЛАН

| Из брифа | В медиаплан |
|---|---|
| PPTX с несколькими кампаниями | требует выбора релевантного блока перед расчётом |
| OLV + баннеры | делится на Video и Display |
| Smart TV ≤ 15% | становится ограничением на долю устройства |
| Бенчи CPM/CPC | используются как контрольные диапазоны ставок |

## 4. Что медиапланер добавил сам

- В медиаплане выбран блок “Инаут”, не обязательно первый блок PPTX.
- Добавлены платформы и таргетинги по продуктовым неделям/сегментам.
- Используются коэффициенты сезонности/гео/частоты/формата.
- Есть связь с фактом прошлой кампании — важный источник benchmark-data.

## 5. Правила, которые надо вынести в MVP

- Для PPTX сначала сегментировать документ на отдельные campaign blocks.
- Если бриф содержит несколько кампаний — пользователь должен выбрать одну или система должна сопоставить по названию.
- Device cap вроде Smart TV ≤15% должен быть hard constraint.
- Бенчи CPM/CPC использовать для проверки плана и подсветки отклонений.

## 6. Что это значит для автоматизации

Для этой пары система должна уметь:

- классифицировать тип кампании;
- выбрать подходящий Excel-шаблон;
- перенести явные вводные из брифа в структуру медиаплана;
- отделить факты от экспертных допущений;
- подсветить недостающие данные;
- сгенерировать черновик, который медиапланер может быстро проверить и поправить.
